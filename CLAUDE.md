## What AI Assistants Should and Should Not Do

### Safe to modify directly
- `styles.css` ‚Äî custom CSS additions or fixes
- `index.html` and other `.html` files ‚Äî direct content or markup edits
- `CNAME` ‚Äî domain changes
- `manifest.webmanifest` / `site.webmanifest` ‚Äî PWA metadata
- `img/`, `media/` ‚Äî add or replace images

### Do NOT modify
- `css/wowchemy.*.css`, `css/vendor-bundle.*.css` ‚Äî minified theme bundles; regenerated by Hugo build
- `js/*.min.*.js` ‚Äî minified bundles; regenerated by Hugo build
- `index.json`, `index.xml`, `sitemap.xml` ‚Äî auto-generated; regenerated by Hugo build
- `webfonts/` ‚Äî Font Awesome assets; should not be hand-edited

### Adding new content
Since this is the compiled output, **new pages require adding the fully rendered HTML**. There is no template system available here. To add a new post or page:
1. Create the directory (e.g., `post/my-new-post/`)
2. Create `index.html` inside it with full HTML (following the pattern of existing post pages)
3. Manually add an entry to `index.json` and `index.xml` for search/RSS
4. Add a link to `sitemap.xml`

> Ideally, new content should be authored in the Hugo source repo and the output regenerated. Hand-editing compiled output is a workaround for when the source is not available.
